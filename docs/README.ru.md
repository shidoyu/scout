# scout — Веб-поиск и получение содержимого страниц

Встроенный WebSearch в Claude Code возвращает фрагменты по 125 символов и опирается исключительно на совпадение ключевых слов. scout преобразует расплывчатый вопрос в оптимизированные запросы для нескольких поисковых систем, оценивает качество результатов и при необходимости выполняет повторный поиск — быстрее и надёжнее добираясь до первоисточников.

## Возможности

- **scout:search** — Многодвигательный веб-поиск с оптимизацией запросов
- **scout:fetch** — Получение содержимого URL с учётом конфиденциальности при выборе инструмента

## Установка

Выполните в терминале:

```bash
# Шаг 1: Зарегистрируйте маркетплейс
claude plugin marketplace add shidoyu/scout

# Шаг 2: Установите плагин
claude plugin install scout@shidoyu-scout
```

## Быстрый старт

scout работает сразу после установки — для поиска используются WebSearch (встроенный) и Exa (бесплатно, ключ не нужен). Дополнительная настройка расширяет возможности:

```bash
bash tools/setup.sh
```

## Skills

### scout:search

Интеллектуальный веб-поиск с:
- Предварительным исследованием для уточнения запросов
- Формированием запросов на нескольких языках
- Несколькими поисковыми системами (WebSearch, семантический поиск [Exa](https://exa.ai))
- HyDE ([Hypothetical Document Embeddings](https://arxiv.org/abs/2212.10496)) для концептуальных запросов через Exa
- Оценкой качества с автоматическим циклом повторного поиска

Использование: `/scout:search ваш вопрос`

### scout:fetch

Получение содержимого веб-страниц с автоматической классификацией по уровню конфиденциальности:
- **Публичные страницы** → Jina Reader (требуется API-ключ) / WebFetch (встроенный запасной вариант)
- **Конфиденциальные страницы** → локальный Playwright (без обращений к внешним API)
- **Страницы с аутентификацией** → Chrome DevTools (сессия браузера)

Использование: `/scout:fetch URL`

## Настройка (необязательно)

Запустите `tools/setup.sh` для настройки:

1. **Exa** — Продвинутые инструменты AI-поиска (API-ключ для платных функций; бесплатный уровень работает без настройки)
2. **Jina Reader** — Высококачественное получение веб-страниц в формате Markdown (требуется API-ключ; без него публичные страницы используют WebFetch)
3. **Playwright** — Получение страниц через браузер для JavaScript-рендеринга и конфиденциального содержимого (~200 МБ загрузки)

Все шаги можно пропустить. Запускайте повторно в любое время для обновления настроек.
После настройки перезапустите Claude Code (или выполните `/mcp`), чтобы новые MCP-серверы вступили в силу.

## Конфиденциальность

scout классифицирует URL по трём уровням перед получением содержимого:
- **Публичные** → облачные API (Jina Reader / WebFetch)
- **Конфиденциальные** → только локальный Playwright (предназначенная маршрутизация: конфиденциальные URL не отправляются во внешние API)
- **С аутентификацией** → Chrome DevTools (использует вашу сессию браузера)

Классификация выполняется автоматически, но основана на суждении LLM, а не на системном принуждении. Подробности — в разделе [Отказ от ответственности в отношении конфиденциальности](#отказ-от-ответственности-в-отношении-конфиденциальности).

## Требования

- Claude Code
- `jq` (только для скрипта установки)
- `npm`/`npx` (для [MCP](https://modelcontextprotocol.io/)-сервера: chrome-devtools)
- Python 3.10+ (необязательно, для локального получения страниц через Playwright)
- `uvx` или `uv` (необязательно, для MCP-сервера: markitdown — конвертация HTML→Markdown)
- Chrome (необязательно, для получения страниц с аутентификацией через DevTools)

### Настройка Chrome DevTools (для страниц с аутентификацией)

Для получения страниц, требующих входа (OAuth, SaaS-дашборды), Chrome должен быть запущен в режиме отладки:

```bash
# macOS
open -a "Google Chrome" --args --remote-debugging-port=9222

# Linux
google-chrome --remote-debugging-port=9222
```

## Отказ от ответственности в отношении конфиденциальности

scout классифицирует URL по уровню чувствительности и направляет конфиденциальные URL только к локальным инструментам.
Классификация основана на суждении LLM (шаблоны доменов и контекст) и **не является гарантией, обеспечиваемой системой**.
При работе с особо чувствительными данными проверьте классификацию перед тем, как продолжить.

**Профиль браузера.** Инструмент получения страниц на основе Playwright (`fetch-page.py`) использует постоянный профиль браузера (`tools/.chrome-profile/`), в котором могут накапливаться файлы cookie, данные сессий и история посещений. Этот каталог исключён из Git через `.gitignore`, однако может быть скопирован инструментами резервного копирования или облачными службами синхронизации. Периодически удаляйте этот каталог, если вы получаете конфиденциальные страницы.
