# scout — Веб-пошук і завантаження вмісту

Вбудований WebSearch у Claude Code повертає фрагменти по 125 символів і покладається лише на пошук за ключовими словами. scout перетворює нечітке запитання на оптимізовані запити до кількох пошукових систем, оцінює якість результатів і за потреби виконує повторний пошук — швидше та надійніше знаходячи першоджерела.

## Можливості

- **scout:search** — Веб-пошук через кілька пошукових систем з оптимізацією побудови запитів
- **scout:fetch** — Завантаження вмісту за URL з вибором інструменту з урахуванням конфіденційності

## Встановлення

Виконайте в терміналі:

```bash
# Крок 1: Зареєструйте маркетплейс
claude plugin marketplace add shidoyu/scout

# Крок 2: Встановіть плагін
claude plugin install scout@shidoyu-scout
```

## Швидкий старт

scout починає працювати одразу після встановлення — пошук використовує WebSearch (вбудований) та Exa (безкоштовно, ключ не потрібен). Додаткове налаштування розширює можливості:

```bash
bash tools/setup.sh
```

## Навички

### scout:search

Інтелектуальний веб-пошук із:
- Попереднім дослідженням для уточнення запитів
- Побудовою запитів кількома мовами
- Кількома пошуковими рушіями (WebSearch, семантичний пошук [Exa](https://exa.ai))
- HyDE ([Hypothetical Document Embeddings](https://arxiv.org/abs/2212.10496)) для концептуальних запитів через Exa
- Оцінкою якості з автоматичним циклом повторного пошуку

Використання: `/scout:search ваше запитання тут`

### scout:fetch

Завантаження вмісту веб-сторінки з автоматичною класифікацією конфіденційності:
- **Публічні сторінки** → Jina Reader (потрібен API-ключ) / WebFetch (вбудований резервний варіант)
- **Конфіденційні сторінки** → Локальний Playwright (без зовнішніх API-викликів)
- **Сторінки з автентифікацією** → Chrome DevTools (сеанс браузера)

Використання: `/scout:fetch URL`

## Налаштування (необов'язково)

Запустіть `tools/setup.sh` для конфігурації:

1. **Exa** — Розширені AI-нативні інструменти пошуку (API-ключ для платних функцій; безкоштовний рівень працює без налаштування)
2. **Jina Reader** — Якісне завантаження веб-сторінок у форматі Markdown (потрібен API-ключ; без нього публічні сторінки переключаються на WebFetch)
3. **Playwright** — Завантаження через браузер для сторінок із JavaScript-рендерингом та конфіденційних сторінок (завантаження ~200 МБ)

Усі кроки можна пропустити. Запускайте повторно будь-коли для оновлення налаштувань.
Після налаштування перезапустіть Claude Code (або виконайте `/mcp`), щоб нові MCP-сервери стали доступними.

## Конфіденційність

scout класифікує URL-адреси за трьома рівнями перед завантаженням:
- **Публічні** → Хмарні API (Jina Reader / WebFetch)
- **Конфіденційні** → Лише локальний Playwright (за задумом: конфіденційні URL не надсилаються до зовнішніх API)
- **З автентифікацією** → Chrome DevTools (використовує ваш сеанс браузера)

Ця класифікація виконується автоматично, але ґрунтується на висновках LLM, а не на технічних гарантіях системи. Докладніше: [Застереження щодо конфіденційності](#застереження-щодо-конфіденційності).

## Вимоги

- Claude Code
- `jq` (лише для скрипту налаштування)
- `npm`/`npx` (для [MCP](https://modelcontextprotocol.io/)-сервера: chrome-devtools)
- Python 3.10+ (необов'язково, для локального завантаження через Playwright)
- `uvx` або `uv` (необов'язково, для MCP-сервера: markitdown — конвертація HTML→Markdown)
- Chrome (необов'язково, для завантаження сторінок з автентифікацією через DevTools)

### Налаштування Chrome DevTools (для сторінок з автентифікацією)

Щоб завантажувати сторінки, які потребують входу (OAuth, SaaS-панелі), Chrome має бути запущений у режимі налагодження:

```bash
# macOS
open -a "Google Chrome" --args --remote-debugging-port=9222

# Linux
google-chrome --remote-debugging-port=9222
```

## Застереження щодо конфіденційності

scout класифікує URL-адреси за рівнем чутливості та направляє конфіденційні URL лише до локальних інструментів.
Ця класифікація ґрунтується на висновках LLM (шаблони доменів і контекст) і **не є гарантією, що забезпечується системою**.
Для особливо чутливих даних перевіряйте класифікацію перед виконанням.

**Профіль браузера.** Засіб завантаження на основі Playwright (`fetch-page.py`) використовує постійний профіль браузера (`tools/.chrome-profile/`), у якому можуть накопичуватися файли cookie, дані сеансів та історія перегляду. Цей каталог виключено з Git через `.gitignore`, але резервні інструменти або служби хмарної синхронізації можуть його скопіювати. Періодично видаляйте каталог, якщо ви завантажуєте конфіденційні сторінки.

## Відмова від відповідальності

Цей плагін надається «як є» відповідно до ліцензії MIT, без будь-яких гарантій.

**Зовнішні API.** Цей плагін покладається на сторонні API (Exa, Jina AI та інші). Автор не надає жодних гарантій щодо доступності, точності, ціноутворення або безперервності цих сервісів і не несе відповідальності за витрати, понесені в результаті використання API.

**Управління API-ключами.** Ви несете виключну відповідальність за отримання, захист і керування власними API-ключами, а також за дотримання умов надання послуг кожного провайдера.

**Класифікація вмісту.** Під час завантаження веб-вмісту плагін може використовувати LLM-класифікацію для оцінки конфіденційності та визначення відповідних методів отримання даних. Такі класифікації є найкращою спробою і можуть містити помилки. Не покладайтеся на автоматичну класифікацію як єдиний захист для конфіденційної або секретної інформації.

**Завантаження веб-сторінок і автоматизація браузера.** Цей плагін містить інструменти для автоматизації безголового браузера через Playwright і Chrome DevTools. Ви несете відповідальність за те, щоб ваше використання відповідало умовам надання послуг цільових вебсайтів, правилам robots.txt та застосовному законодавству. Автор не несе відповідальності за блокування сайтів, призупинення облікових записів, обмеження IP, неочікуване виконання скриптів, споживання ресурсів або проблеми сумісності, що виникають внаслідок автоматизації браузера.

**MCP-сервери.** Цей плагін підключається до сторонніх MCP (Model Context Protocol) серверів. Автор не контролює, не перевіряє і не гарантує поведінку або безпеку цих серверів.
